# 번트(BUNT) 프로젝트 개발 규칙

## 1. 실전 매매 코드 변경 시 필수 검증

실제 자금이 걸린 시스템이다. 추측으로 "됩니다"라고 답하지 않는다.

### 변경 전
- 변경 대상 함수뿐 아니라, 동일 패턴이 적용된 **모든 함수를 전수 점검**한다
- 예: 매수 파이프라인 수정 시 → 매도, 체결확인, 리트라이, API 호출 전체 점검

### 변경 후 (배포 전 필수)
- **반드시 테스트 코드를 작성하고 실행**한다 (node -e 또는 테스트 스크립트)
- KIS API 호출이 포함된 변경은 **실제 API를 호출하여 검증**한다
- 토큰 발급/갱신 변경 시 → 토큰 발급 → 현재가 조회 → 잔고 조회 순서로 검증
- 단위 변환(소수↔퍼센트, 원↔만원) 변경 시 → 해당 값을 사용하는 **모든 출력(로그, 이메일, 대시보드)을 점검**

### 절대 금지
- 테스트 없이 "수정 완료"라고 답하는 것
- 서버 상태를 확인하지 않고 추측으로 답하는 것
- 한 곳만 고치고 동일 패턴의 다른 곳을 누락하는 것

## 2. KIS API 주의사항

- 토큰 발급은 **1분당 1회 제한** (rate limit). 무조건 재발급하면 안 됨
- 토큰 유효성 검증: 현재가 조회(가벼운 API)로 실검 → 무효할 때만 재발급
- 모의/실전 토큰은 **별도 파일로 분리** (token-paper.json, token-real.json)
- 장운영시간 외에는 주문 API 거부됨 (09:00~15:30)

## 3. 매도는 생명줄

- 매도 실패 = 손실 확대. 매도 관련 코드는 **최우선으로 안전장치** 확보
- 매도 파이프라인: 토큰 실검 → 주문 → 실패 시 즉시 재시도(2회) → 1분 리트라이(10회) → 이메일 알림
- 매도 코드 변경 시 **기존 안전장치가 깨지지 않았는지 반드시 확인**

## 4. 서버 프로세스 확인 방법

```bash
# 번트 서버 프로세스 확인
pgrep -f "node.*번트/server.js"

# 로그 확인 (당일)
cat logs/bunt-$(date +%Y-%m-%d).log

# 서버 상태 API
curl -s http://localhost:3100/api/status
```

## 5. 프로젝트 구조

- `server.js` — 메인 서버, 크론 스케줄, 매수/매도 파이프라인
- `scanner.js` — 네이버 API 종목 스캔 (파이프라인 1~2단계)
- `dart-filter.js` — DART Open API 악재 필터 (파이프라인 2단계 ⑦)
- `account-service.js` — KIS API 래퍼 (토큰, 주문, 잔고, 현재가)
- `email-service.js` — 매매 알림 이메일
- `data/config.json` — 설정 (API키 포함, .gitignore 대상)
- `data/state-paper.json` / `state-real.json` — 모드별 상태
- `data/dart-cache.json` — DART 조회 결과 당일 캐시
- `data/dart-corp-map.json` — 종목코드→DART 기업코드 매핑 (30일 TTL)
- `engine/` — 백테스트 엔진 (분석 전용, 서버 영향 없음)

## 6. 엔진 로직 (전체 파이프라인)

### 전략 요약
전일 장마감 직전 상한가 종목 매수 → 다음날 장 시작 시가 매도 (오버나잇 갭업)

### [1단계] 종목 스캔 (15:15~15:20)
- 네이버 모바일 API → 코스닥 전종목(~1,820개) 등락률 리스트 조회
- 등락률 상위 30종목 → 개별 OHLCV(시가/고가/저가/종가) 상세 조회

### [2단계] 필터링 (7단계 순차)
| 순서 | 필터 | 기준 | 처리 위치 |
|------|------|------|-----------|
| ① | 신규상장 제외 | 등락률 > 30% | scanner.js |
| ② | 관리종목 제외 | 블랙리스트 | scanner.js |
| ③ | 고정 상한가 제외 | 시가=고가=종가 | scanner.js |
| ④ | 등락률 필터 | ≥ 29% | scanner.js |
| ⑤ | 종가=고가 필터 | 종가/고가 = 100% | scanner.js |
| ⑥ | 연속 재진입 필터 | 전일 매수 종목 제외 | server.js |
| ⑦ | DART 악재 필터 | 연속적자(3기중 2기+) / 악재공시(90일) | dart-filter.js |

→ 최대 10종목 선정 (등락률 내림차순)

### [3단계] 분할 매수 (50/50)
| 단계 | 모의 | 실전 | 방식 |
|------|------|------|------|
| 1차 | 15:05 | 15:06 | 정규장 시장가 (50%) |
| 2차 | 15:21 | 15:22 | 장마감 동시호가 (50%) |

### [4단계] 오버나잇 보유

### [5단계] 매도 (다음날)
| | 실전 | 모의 |
|--|------|------|
| 시간 | 08:50 | 09:01 |
| 매도가 | 시장가 주문 → 09:00 동시호가(시가) 체결 | KIS API 시가(open) 조회 → 시가 정산 |
| 매수가 | 체결확인에서 실제 체결가(avgPrice) 반영 | 전일 종가(prevClose)로 보정 |

- 실전 매도 안전장치: 즉시 재시도(2회) → 프로그레시브 리트라이(30초~10분, 5단계) → 이메일 알림
- 모의/실전 수익률 괴리: 0%p (시가 정산 + 매수가 보정)

### [6단계] 정산
| 비용 항목 | 요율 |
|----------|------|
| 매수 수수료 | 0% (국내수수료우대 계좌) |
| 매도 수수료 | 0% |
| 매도세 (코스닥) | 0.20% (증권거래세) |
| **왕복 합계** | **0.20%** |
| 비고 | 실전: KIS API output2에서 실제 수수료/세금 조회 |
